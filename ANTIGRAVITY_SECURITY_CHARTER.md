# ANTIGRAVITY SECURITY CHARTER (アンチグラビティ・セキュリティー憲章)

## 概要
この文書は、Anti-Gravity プロジェクトにおけるすべてのソフトウェア開発において、機密情報の漏洩を未然に防ぎ、透明性の高い開発環境を維持するための絶対的な指針である。

## 1. 秘匿情報の厳禁 (Absolute Secret Shield)
- **ハードコードの禁止**: APIキー、パスワード、トークンなどの機密情報を、いかなる場合もソースコード内にリテラル（文字列）として直接記述してはならない。
- **一時的な埋め込みの排除**: 「テストのためだけ」「あとで消す」は許されない。最初から環境変数として扱う。

## 2. 環境変数の集中管理 (Centralized Config)
- **単一の入り口**: 各プロジェクトには、環境変数を一括で管理し、バリデーションを行う `config.ts`（あるいはそれに準ずるモジュール）を必ず設ける。
- **存在確認の自動化**: アプリケーション起動時に必要な環境変数が不足している場合、即座にエラーを出してプロセスを停止させる（フェイルファスト）。

## 3. 監視と監査 (Continuous Audit)
- **.gitignore の徹底**: `.env` ファイルや機密情報を含む可能性のあるディレクトリ（`.vercel`, `.expo` など）は、プロジェクト開始時に必ず無視リストに登録する。
- **AIエージェントの自律監査**: 私（AIエージェント）は、プッシュやデプロイの前に、コード内に機密情報が混入していないか自律的にチェックし、疑わしい場合は作業を中断する。

## 4. 事故対応 (Response Protocol)
- **即時通知**: 万が一、流出の疑いがある場合は、直ちにユーザーに通知し、キーの無効化を促す。
- **完全消去**: Gitの履歴を含め、流出した情報を物理的に消去するための技術的手段を講じる。
